# ADR-002 — 台灣詞彙映射策略（TaiwanLexicon）

> 狀態：已採納  
> 日期：2026-04-01  
> 決策者：Johnny Lu  

---

## 背景

FR-01 要求台灣特有詞彙與發音的 LEXICON 映射，覆蓋率目標 ≥ 95%。詞彙類別包括：
- 科技詞（視頻→影片）
- 交通詞（地鐵→捷運）
- 日常詞（垃圾→ㄌㄜˋ ㄙㄜˋ）
- 食物詞（菠蘿→鳳梨）
- 職業詞（程序員→工程師）
- 發音詞（和→ㄏㄢˋ、吧→啦）

---

## 決策

**採用 Hash-based String Replacement 映射表**，不放棄以下任一方案。

### 替代方案分析

| 方案 | 優點 | 缺點 | 結論 |
|------|------|------|------|
| **Hash 映射表** | O(1) 查詢、簡單維護、≥50 詞可達成 | 需維護詞表 | ✅ 採用 |
| ML/NLP 模型 | 可處理未知詞 | 延遲高、過度複雜 | ❌ 放棄 |
| 規則引擎 | 可擴展 | 過度工程，詞彙替換不需要完整規則引擎 | ❌ 放棄 |

---

## 理由

1. **簡單有效**：字串替換在文本進入 TTS 前執行，無額外延遲（O(n) 線性掃描）
2. **可測試性**：每個詞彙都有明確的 input/output 映射，單元測試容易覆蓋
3. **詞彙量門檻**：SRS 要求 ≥ 50 詞，Hash 表可簡單擴展至 100+ 詞
4. **優先於 NLP**：TTS 合成延遲要求（TTFB < 300ms），ML 模型不適用

---

## 設計

```python
# engines/taiwan_linguistic.py

TAIWAN_LEXICON: dict[str, str] = {
    # 科技詞
    "視頻": "影片",
    "軟件": "軟體",
    # 交通詞
    "地鐵": "捷運",
    # 日常詞
    "垃圾": "ㄌㄜˋ ㄙㄜˋ",
    # 食物詞
    "菠蘿": "鳳梨",
    # 更多詞...
}
```

**擴展策略**：
- 按類別分組（科技、交通、食物等）
- 每個類別獨立的 `_category_lexicon` 子字典
- 主要映射表合併所有子字典

---

## 結果

### 正面
- 查詢時間 O(1)（Hash 表）
- 易於版本化（Git 追蹤詞表變更）
- 單元測試覆蓋簡單

### 負面
- 無法處理上下文相關的歧義（e.g., "程序" 在 "程序員" vs "計算程序"）
- 長詞優先匹配需確保替換順序

---

## 驗證

- FR-01：`TAIWAN_LEXICON` 總詞彙數 ≥ 50
- NFR-02：LEXICON 覆蓋率 ≥ 80%（目標 ≥ 95%）
- NFR-03：變調正確率 ≥ 95%（詞彙映射後單元測試驗證）
