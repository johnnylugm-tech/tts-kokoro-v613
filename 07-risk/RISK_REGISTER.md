## 2. 風險詳細說明

### R-01：Kokoro Docker 崩潰導致 TTS 服務中斷

| 欄位 | 內容 |
|------|------|
| **ID** | R-01 |
| **類別** | 外部依賴 / 技術風險 |
| **影響** | 高 — 後端崩潰時整個 TTS 服務不可用 |
| **可能性** | 低 — Docker 容器預期穩定運行 |
| **風險分數** | 12（影響高×可能性低） |
| **根本原因** | Kokoro Docker 是外部單點依賴，無叢集/備援 |
| **觸發條件** | Kokoro 行程崩潰、連接埠 8880 無響應、記憶體不足 |
| **現有緩解** | 斷路器（Circuit Breaker）已實作（FR-05）；30 秒 timeout |
| **殘餘風險** | 斷路後服務回傳 HTTP 503，但不影響系統穩定性 |
| **狀態** | 監控中 |
| **擁有者** | Backend Team |

### R-02：斷路器誤判導致正常請求被阻斷

| 欄位 | 內容 |
|------|------|
| **ID** | R-02 |
| **類別** | 技術風險 |
| **影響** | 高 — 誤判會導致服務降級 |
| **可能性** | 低 — 斷路器邏輯已測試 |
| **風險分數** | 8 |
| **根本原因** | 閾值設定（失敗 ≥ 3 次）可能因網路瞬斷而觸發 |
| **觸發條件** | 快速連續 3 次 5xx（可能是網路瞬斷非後端崩潰） |
| **現有緩解** | Open 後 10 秒自動嘗試恢復（Half-Open）；L4 Mock 測試已驗證 |
| **殘餘風險** | 網路抖動場景下可能短暫斷路 |
| **狀態** | 監控中 |
| **擁有者** | CircuitBreaker |

### R-03：Redis 連線失敗導致快取失效

| 欄位 | 內容 |
|------|------|
| **ID** | R-03 |
| **類別** | 技術風險 / 可選依賴 |
