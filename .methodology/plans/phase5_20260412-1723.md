## 5. 兩次 A/B 審查流程

### 第一次審查（BASELINE，部署前）

```
Agent A (devops) 建立 BASELINE.md + QUALITY_REPORT.md + MONITORING_PLAN.md
         ↓
Agent B (architect) 審查 BASELINE.md
         ↓
     ❌ REJECT → Agent A (devops) 修訂 → 重新審查
         ↓
     ✅ APPROVE → 才能執行部署
```

### 第二次審查（VERIFICATION_REPORT，部署後）

```
Agent A (devops) 執行驗收測試 + 建立 VERIFICATION_REPORT.md
         ↓
Agent B (architect) 審查 VERIFICATION_REPORT.md
         ↓
     ❌ REJECT → Agent A (devops) 修復 → 重新審查
         ↓
     ✅ APPROVE → Quality Gate → sign-off
```

## 8. sessions_spawn.log 格式（HR-10）⚠️ 重要

**⚠️ 即時寫入，不是完成後補**

每次 `sessions_spawn()` 呼叫時，**在派遣前**寫入：

```json
{"timestamp":"ISO8601","role":"devops","task":"任務名稱","session_id":"agent:...","fr":"PHASE5","confidence":0,"verdict":"N/A"}
```

派遣完成後更新 verdict：

```json
{"timestamp":"ISO8601","role":"devops","task":"任務名稱","session_id":"agent:...","fr":"PHASE5","confidence":1-10,"verdict":"PASS|APPROVE|REJECT"}
```
